# 1 项目提出

**项目名称**：基于 DeepSeek 大模型的交通知识培训平台

**项目简介**：

随着我国机动车保有量持续增长（截至2025年已突破4.5亿辆），道路交通安全培训需求日益迫切。传统交通知识培训以线下课堂和纸质教材为主，存在教学形式单一、个性化不足、学习效果难以量化等问题。每年超3000万人参加驾驶证考试，科目一和科目四理论考试通过率仅约70%，大量学员因学习方式不匹配而反复补考。现有驾驶培训平台题库更新滞后、模拟考试与实际脱节、缺乏针对性薄弱点分析。

本项目基于DeepSeek大模型技术，构建智能化、个性化的交通知识培训平台，覆盖交规学习、驾驶证备考、安全驾驶培训、违章学习等多场景，为学员提供从报名到拿证的全流程智能化学习服务。

**项目目标**：

1. 构建覆盖科目一、科目四核心题库（500+题）的智能练习系统，支持按知识点、题型分级练习与AI智能推题
2. 基于DeepSeek大模型实现交规知识AI讲解，提供法规原文引用、典型案例分析和追问能力
3. 建立能力基线测评与个性化学习路径规划机制，根据学员薄弱点动态调整学习计划
4. 实现全真模拟考试功能，随机组卷、限时作答、即时评分
5. 提供驾校管理后台，支持学习进度监控与学员成绩统计
6. 基于RAG技术确保AI回答与现行法规一致，保障回答准确性与合规性

**系统边界**：

系统分为学员端（Web端）与驾校管理端（Web后台）两大入口，聚焦MVP核心功能：

- **学员端**：注册与档案管理、能力基线测评、个性化学习路径规划、智能题库练习、AI知识讲解、智能错题分析、全真模拟考试、学习进度可视化、学习提醒与激励
- **驾校管理端**：学员学习进度查看、模拟考试成绩统计、学员管理
- **智能客服**：基于DeepSeek的客服机器人基础问答

# 4 风险评估和规避

## 技术风险

1. **AI 回答准确性风险**：DeepSeek大模型在回答交通法规问题时可能出现幻觉或错误，若AI回答与现行法规不一致，可能误导学员。

2. **大模型API依赖风险**：核心功能（AI讲解、智能客服）依赖DeepSeek API的可用性，存在服务中断或响应延迟风险。

3. **技术栈学习曲线风险**：3人团队需在短时间内掌握DeepSeek API接入、RAG技术、前后端全栈开发，团队成员技术水平参差不齐可能导致进度受阻。

解决：

1. 采用RAG（检索增强生成）技术，将AI回答约束在官方法规和题库范围内；建立核心问题的预置标准答案兜底机制；对AI生成内容进行人工抽检。
2. 实现API调用降级策略，当DeepSeek API不可用时自动切换至本地缓存的标准解析；对高频问题建立预计算缓存，减少实时API依赖。
3. 项目启动前2天集中进行技术预研与培训，明确技术选型与架构方案；采用成熟稳定的技术栈（如Spring Boot + Vue），降低学习成本；每日站会同步进度，及时解决技术障碍。

## 管理风险

1. **工期极度紧张风险**：总计90人天工作量压缩在30个自然日内完成，平均每人每天需产出3人天有效工作量，进度压力极大。

2. **多角色兼任效率风险**：每位成员均需兼任2-3个角色（如组长同时担任项目经理、DBA、主力程序员），角色频繁切换可能导致注意力分散、工作效率下降。

3. **单人故障点风险**：团队仅3人，任一成员因病或意外缺席将导致其负责模块停滞，缺乏替补人员。

解决：

1. 采用MVP策略，优先交付核心功能（题库练习、AI讲解、模拟考试），次要功能作为可选项；每日制定明确的可交付目标，当日事当日毕；预留最后3天作为缓冲和集中测试修复期。
2. 以半天为单位划分工作时段（上午专注开发、下午处理管理与测试事务），减少角色切换频率；使用TAPD/飞书等协作工具统一任务管理，避免信息分散。
3. 代码每日提交至Git仓库，确保知识资产不集中在单机；核心模块采用结对编程，互相了解对方代码；关键文档（数据库设计、API接口文档）团队共享，确保任一成员可接手他人工作。

## 其它风险

1. **需求变更风险**：实训项目中指导老师或评审方可能在开发过程中提出需求调整，30天工期下需求变更代价极高。

2. **部署与环境风险**：开发环境、测试环境与最终演示环境可能存在差异，导致部署失败或功能异常。

解决：

1. 项目启动前完成需求评审与确认，冻结核心需求范围；非核心需求变更记录至"后续迭代"清单，不在本次30天内实现；确需变更的核心需求须经全组评估工作量后置换同等规模的低优先级功能。
2. 统一使用Docker容器化部署，确保环境一致性；每日进行至少一次集成部署验证；最终演示前3天完成完整部署与全流程回归测试。
